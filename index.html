Why is it good to be accessible

add_task

Enhanced User Experience

Accessibility measures improve the overall user experience for everyone, including individuals with disabilities. Clear navigation, proper color contrast, and readable content benefit all users by making the website or application more intuitive and user-friendly.

dataset

Expanded Reach and Inclusivity

By implementing accessibility features, you ensure that your digital platforms can be accessed and used by a wider audience. This includes individuals with disabilities, seniors, and those using assistive technologies, allowing them to fully engage with your content and services.

token

Legal Compliance and Avoidance of Discrimination

By adhering to accessibility guidelines and regulations, you ensure legal compliance and mitigate the risk of discrimination claims. Accessibility is a legal requirement in many jurisdictions, and by proactively addressing accessibility, you create an inclusive environment and mitigate potential legal issues.

view_timeline

Improved Search Engine Optimization and Search Visibility

Accessible design practices often align with (SEO) guidelines. Implementing accessibility features such as proper heading structure, alt text for images, and descriptive link text not only make your website more accessible but also improve its visibility and ranking in search engine results.

processes to do accessiblity

html code

doing better accessiblity with html

html always use semantic html tags eg: <hr> <main> <aside> <section> and good content or logical content organization
code any deviation from semantic ui controls needs keyboard accessiblity to be built back in into the deviation
integration_instructions use ui labels, tables with a <caption> or summary attribute. always include alt & title attributes for images or use <figure><figcaption> or empty alt for decorative images
developer_mode links should be styled propely according to their states.. that is; ( :hover, :visited, :link ) links should be used for navigation and only navigation( buttons can be used for in-page naviagtion instead )

css data_object

create accessible visuals with css

bad contrast good contrast always employ good WCAG contrast rules in your web design
always make sure you dont style native UI elements too differently from their native looks
either hide content completely from screen reader users or display it fully in the right format
when styling tables, always make the individual sections stand out .. that is; the rows, columns, headings

roles

roles are used to give accessiblity tools information on what element does or what instead function is supposed to be - aria should be used for the enhacement of accessiblity

add_task

properties

these are almost similar to normal html attributes, they give more info on the element - properties should be used to complement additional roles or native roles in elements

add_task

states

these give more info on the internal condition or state of an element ( eg: whether an element is disabled or readonly ) - states used for change or render-based stuff, states change but properties don't

multimedia play_circle

images

always include alt attributes, include title attributes if neccessary. if image is decorative, use no alt attribute. you can also use "aria-labelledby" to give bulk descriptions of images

audio

always include transcripts for your audio. include text tracks if possible. make sure that in your transcripts any reference to documents in the audio are placed therein

video

always include text tracks for your video; whether they're captions, subtitles, descriptions, chapter markers or any others. the benefit of text tracks can't be overemphasized - they help screen readers, impaired people, people watching on low bandwidths..etc
